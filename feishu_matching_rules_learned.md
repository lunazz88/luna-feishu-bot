# 飞书广告数据匹配规则学习结果

样本来源：

- 2026-06-05：广告原始表、投手自填晨会表、Ricky 修正表
- 2026-06-06：广告原始表、投手自填晨会表、Ricky 修正表

## 一、字段含义

广告原始表是指标可信来源：

- 花费 -> 花费金额
- 展示数 -> 展示次数
- 点击数 -> 点击量
- 注册数 -> 注册人数
- 首存数 -> 首存人数
- 注册到首充率、注册成本、首存成本等衍生列应按修正后的指标重新计算或按广告表值格式化

晨会表是投手申报项目来源：

- 项目、Code、投手、国家、KPI 主要用于定位行
- 指标列不可信，匹配成功后应优先用广告原始表修正

## 二、清洗规则

### 项目

1. 去掉晨会表项目名前面的序号。
   - `01(Fine)Winbd-BD` -> `(Fine)Winbd-BD`
   - `26(Fine)Vipganer-MX` -> `(Fine)Vipganer-MX`
2. 统一中英文括号。
3. 去掉多余空格。
4. 项目名是强匹配字段，不能只靠 code 跨项目匹配。

### Code

Code 必须分两层处理，不能一步强清洗。

1. 轻度清洗 code，优先用于精确匹配：
   - 去空格
   - 统一大小写
   - 保留 `code` 字样和前导 0
   - 例如 `01code` 仍是 `01code`
2. 强清洗 code，用于兜底匹配：
   - 去掉任意位置的 `code`
   - 去掉纯数字前导 0
   - `code1501` -> `1501`
   - `code0501` -> `501`
   - `code08` -> `8`
   - `code01` -> `1`
3. 如果同一项目里同时存在 `01code` 和 `1`，必须先走轻度清洗精确匹配，否则会误把两行都归成 `1`。

### 投手

1. 投手不能作为硬性匹配条件。
2. 广告表投手和修正表投手经常不一致，但 Ricky 仍然按项目 + code 修正数据。
   - 例：`(Fine)Bdbajee-BD / 503`，修正表投手是 `Phoebe`，广告表投手是 `Alan`，仍然匹配。
   - 例：`(Fine)Winbd-BD / 1511`，修正表投手是 `Sarah`，广告表投手是 `Phoebe`，仍然匹配。
3. 投手只作为冲突时的辅助确认字段。
4. 如果项目 + code 找不到唯一广告行，可尝试项目 + 投手 + 国家 + KPI 兜底。

## 三、匹配优先级

按以下顺序匹配，命中唯一行才自动修正：

1. 项目 + 轻度清洗 code + 国家 + KPI
2. 项目 + 轻度清洗 code + 国家
3. 项目 + 强清洗 code + 国家 + KPI
4. 项目 + 强清洗 code + 国家
5. 项目 + 强清洗 code
6. 项目 + 投手 + 国家 + KPI
7. 仍不唯一或找不到，输出到人工确认清单

不能只用 code 匹配，因为同一个 code 会在多个项目重复出现。

## 四、样本验证结果

2026-06-05：

- 广告表 453 行
- 自填表 456 行
- Ricky 修正表 421 行
- 修正表中 417 行可按规则匹配广告表
- 4 行未匹配，均为 `(Fine)Ggoopg-BR`

2026-06-06：

- 广告表 452 行
- 自填表 456 行
- Ricky 修正表 422 行
- 修正表中 418 行可按规则匹配广告表
- 4 行未匹配，均为 `(Fine)Ggoopg-BR`

两个日期里，自动匹配覆盖率约 99%。

## 五、行保留和删除规则

1. 晨会表中的空白分隔行会被删除。
2. 自填表中存在但广告表没有对应项目/code 的行，不能直接写 0，应进入人工确认。
3. 样本里 `14(Fine)Winpkr-PK` 的 `tech01` 到 `tech04` 在 Ricky 修正表中被删除，说明 `tech` 类 code 可能不是广告原始表的有效匹配项，需要单独规则或人工确认。
4. `(Fine)Ggoopg-BR` 在两天修正表中都保留但广告表未匹配到，应作为“广告未抓取但人工保留”的典型例外。

## 六、指标修正规则

1. 匹配成功后，花费、展示、点击、注册、首存原则上取广告表。
2. 金额类按美元格式保留两位或按原表格式显示。
3. 百分比类按修正后的注册、首存、展示、点击重新计算。
4. 广告表为 0 时不能简单全部写 0：
   - 有些行广告花费/展示/点击为 0，但注册/首存有值，Ricky 修正表会保留注册/首存。
   - 有些行广告花费/展示/点击为 0，但修正表保留了人工补充的展示/点击，例如 `(Fine)Winbd-BD / code15`。
5. 因此建议自动流程中：
   - 广告表非 0 的指标直接覆盖。
   - 广告表注册/首存非 0 时直接覆盖。
   - 广告表花费/展示/点击为 0 但自填表或历史规则有非 0 时，输出“0 值冲突待确认”，不要静默覆盖。

## 七、飞书自动化建议

每次处理时生成 4 个结果区：

1. 已自动匹配并修正
2. code 强清洗后匹配
3. 投手不一致但项目 + code 匹配
4. 未匹配或 0 值冲突，等待人工确认

这样能复刻 Ricky 的主要人工规则，同时避免把不确定行误修。
